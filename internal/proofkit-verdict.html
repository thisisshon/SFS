02 Feature-by-feature

The reality check

Each feature mapped to what the code actually does today. Filter by grade; the amber cards are where the spec and the codebase disagree.

⚠ The four the spec gets wrong

These read as greenfield in the spec but already exist in the code — building them blind means duplicating work or missing a prerequisite. Re-scope before you schedule them.

F5 Pin colours

"Reads existing status field" — but pins read the dead status field, not the Worker's teamStatus. Rewire first, then it's a pure paint.

F10 Location hint

~80% built: cssPath() already emits a selector preferring data-cms/id. Mostly a display hint, not capture work.

F3 Reopen reasons

Reopen already requires + stores a reason. This is a free-text → enum + conditional-note upgrade, not new plumbing.

F6 Quick replies

Overlaps the existing parentId reply threads. Decide replace-vs-parallel before building a second structure.
